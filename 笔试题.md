## 笔试题 1：全栈 AI 工程实战复盘

### 题目

用一个你**真实经历**的项目，展示你的全栈 AI 工程能力。

**要求：**
- 必须包含 **Agent 开发**技术
    
- 可选再包含以下其他技术栈， 
    可选类别：
    - [ ] AI/ML/DL（特征工程/训练/推理/评估/部署）
    - [ ] 前端（React/Vue/Angular/Svelte/可视化）
    - [ ] 后端（Python/Java/Go/Node/Rust/FastAPI/Django）
    - [ ] 基础设施（Linux/Shell/Bash/脚本自动化）
    - [ ] 容器化（Docker/K8s/Compose/Helm/CI/CD）
    - [ ] 数据库（MySQL/PostgreSQL/Redis/MongoDB/ClickHouse）
    - [ ] 数据工程（Pandas/Polars/Spark/ETL/可视化）
    - [ ] 并行计算（Ray/Dask/Spark/多进程/多线程/GPU 并行）
    - [ ] 消息队列（Kafka/RabbitMQ/Redis Streams/Celery）
    - [ ] 监控运维（Prometheus/Grafana/ELK/Loki/Sentry）
- 描述一个或多个**具体的项目**及其完整的排查和解决过程
- 必须真实，禁止 AI 写作, 检测到AI写作扣分50%，字数不作为考虑，内容占比重

**答题结构：**
1. **项目背景**：项目是什么，解决什么问题，使用场景是什么（可脱敏）。
2. **个人工作**：你负责的部分是什么，做了哪些事情，产出是什么。
3. **Agent 目标**：Agent 要完成什么任务，输入/输出是什么，成功标准是什么。
4. **方案与流程**：你的整体方案是什么，主要流程是怎样的。
5. **Prompt**：你最关键的 Prompt（或模板）是什么，分析一下设计逻辑。
6. **个人经验**：纯 LLM 直接调用在 Agent 场景的主要缺陷是什么？你会如何优化与增强 Agent 的可靠性与可控性？
7. **验证与迭代（可选）**：你如何验证有效，做过哪些迭代与改进；可选写你如何让它稳定运行或如何做最小可用版本。
8. **复盘与反思（可选）**：你从这个项目中学到的 3 个关键经验是什么？下次你会优先改哪 1 件事？

**量化证据清单（强烈建议提供，影响评分）**：
- 至少提供 6 条可核验证据（建议放在文末证据区）
- 可核验证据类型示例：
  - 错误日志/错误码/stack trace
  - 关键配置项及取值（如超参、连接池、超时时间）
  - 版本信息（模型版本、框架版本、依赖版本）
  - 性能数据前后对比（如时延、吞吐、成本、成功率）
  - 关键命令或脚本（如排查命令、复现命令）
  - Prompt 迭代记录（至少 2 版）与对应效果变化
  - 失败尝试与回滚记录
  - 明确时间线（问题发现 -> 排查 -> 修复 -> 验证）

**技术栈类答题结构（可选）**（以下为可参考模板）：

- **技术目标**：你要解决的具体工程问题是什么？
- **方案选择**：你对比过哪些方案？为什么最后选这个？
- **架构与边界**：你负责的模块在系统中的位置是什么？它与上下游如何交互？
- **关键实现**：你做了哪些关键设计与实现？有哪些关键约束与取舍？
- **验证结果**：你如何评估产出？
- **问题与排查**：你遇到过的一个典型问题是什么？你如何debug与修复？
- **与 Agent 的结合/优化**：它可以如何与 Agent 结合？它如何优化 Agent 的效果或体验？


## 笔试题 2：Agent 驱动的金融建模系统

### 题目

使用 **AI Agent 系统**完成金融时序数据的建模任务。**从以下两个方向任选其一**。

**数据：**
- 数据文件：`data.pq`（与本题同包提供，解压即得）
- 基础字段：`trade_date`, `underlying`, `start_time`, `end_time`, `open`, `high`, `low`, `close`, `volume`
- 特征字段：`X1` ~ `X300`
- 标签字段：`Y1` ~ `Y12`

---

### 方向 A：自动化特征工程系统

**任务**：Agent 系统自动从 300 个特征中筛选最优 50 个特征，完成以下流程：

1. **自动特征诊断**：Agent 遍历所有特征，自动检测缺失值、异常值、分布特征，并生成诊断报告。
2. **自动特征清理**：Agent 针对每个特征自动决策处理方案，生成并执行清理代码。
3. **自动特征评估**：Agent 自动计算特征有效性，检测特征冗余。
4. **自动特征筛选**：Agent 多轮迭代筛选输出 Top 50 特征及理由。
5. **输出可视化**：每个特征的处理记录、Top 50 特征分析。

---

### 方向 B：自动化模型选择系统

**任务**：Agent 系统自动尝试多种模型并选择最优方案，完成以下流程：

1. **自动数据预处理**：Agent 分析数据特点，自动决策预处理方案（标准化/归一化/PCA/特征筛选），生成并执行代码。
2. **自动模型选择**：Agent 自动训练多种模型（至少 3 类）：
   - 线性模型（Logistic/SVM）
   - 树模型（XGBoost/LightGBM/CatBoost）
   - 神经网络（MLP/LSTM/Transformer）
   - 自定义有效深度学习模型架构（加分项）
3. **自动模型评估**：Agent 自动计算分类指标，对比模型性能。
4. **结果分析**：对最优模型分析性能表现，生成模型对比报告。
5. **输出可视化**：数据预处理效果、所有模型对比、最优模型性能曲线、混淆矩阵等。

**注**：方向 B 需要完成分类任务，分类标签从 `Y1~Y12` 中自选。

---

### 共同要求（两个方向都需要）

**硬性指标**：
- 分类标签：从 `Y1~Y12` 中自选
- 分类指标：AUC/Precision/Recall/F1（必须计算）
- 可视化：数据概况、训练/评估、性能曲线（混淆矩阵/ROC 曲线等）
- 数据泄漏检测：无未来数据用于训练

**核心要求**：
- 设计并实现一个 **Agent 系统**（可使用 AI 辅助）
- Agent 需要自主决策、自动生成代码、自动执行、自动生成报告
- 提交可运行的代码与材料（不限字数，可使用 AI 辅助）

**提交内容**：
1. **Agent 系统设计文档**：架构图、完整 Prompt、工作流设计、异常处理机制
2. **代码与运行入口**：完整代码 + README + 可运行的 `.ipynb`（一键复现 Agent 执行过程）
3. **运行日志与结果**：Agent 的完整输出（分析/决策/生成的代码）、执行结果（可视化/指标/报告）

**重要提示**：
- `.ipynb` 必须是**已经运行过的版本**，所有 code cell 必须包含 `execution_count` 和完整输出结果
- 不接受未运行的空白 Notebook 或仅有代码无输出的文件
- 评分时会检查输出的完整性，缺失输出将严重扣分
- 建议在 Notebook 中增加“结果索引”Markdown，明确标注以下关键输出所在 cell：
  - 数据概况输出
  - 训练过程输出
  - 评估指标输出（AUC/Precision/Recall/F1）
  - 可视化输出（混淆矩阵/ROC 等）
  - 数据泄漏检查输出

**提交：**
- 时间：收到数据后 48 小时
- 提交地址：https://www.ten-x-ai.com/interview/submit-assignment
